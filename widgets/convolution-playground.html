Region Processing · Convolution

Nine numbers,
any effect.

Blur, sharpen, edge-detect, emboss — every one is the same operation: slide a 3×3 grid of weights over the image and replace each pixel with the weighted sum of its neighbours. Change the nine numbers, change everything.

Image Processing the heart of Week 2+

The one equation behind all of it HIGH-YIELD

I′(x,y) = Σi,j  K(i,j) · I(x+i, y+j)   // i,j ∈ {−1,0,+1}

Unlike point processing (which sees one pixel), convolution is region processing: the new pixel depends on its neighbours. The 3×3 grid of weights K is the kernel (a.k.a. mask / filter). That's the whole idea — the rest is choosing the nine numbers.

What each kernel does, and why

Blur (averaging)

All-positive weights that sum to 1 (after normalising). Each pixel becomes a mix of its neighbours → high-frequency detail smeared out. Gaussian weights the centre more, for a softer, more natural blur than the box.

Use: noise reduction, pre-smoothing before edge detection.

Sharpen

Centre weight > 1, negative neighbours, summing to 1. It amplifies the difference between a pixel and its surroundings — edges get exaggerated. Push it too far and you get halos + clipping.

Use: enhancing detail.

Edge detection

Weights sum to 0. On flat regions the sum cancels to 0 (black); only where intensity changes does it fire. Sobel X/Y respond to vertical/horizontal edges; Laplacian to edges in all directions.

Use: feature extraction (the CV pipeline's step 2).

🧠 Hook — read the kernel's sum

Sum = 1 → brightness preserved (blur, sharpen). Sum = 0 → flat areas go black, only edges survive (edge detectors). That single check tells you what a mystery kernel will roughly do before you run it.

Exam framing

What gets asked

"Apply this 3×3 kernel to the centre pixel of the following neighbourhood" — a hand-computation. Method: line the kernel up, multiply each weight by the pixel under it, sum the nine products, divide by the kernel sum if normalising, then clip to [0,255]. Hover the playground to see exactly this arithmetic, then reproduce it on paper.

EDGE OF THE IMAGE

At the border, part of the 3×3 window falls outside the image. Common fixes: replicate the edge pixel (used here), zero-pad, or simply skip the border. State which one you assume — examiners accept any, but want you to acknowledge the problem exists.
