Image Processing Ch 1–2 · point processing

Two equations, both already in your Chapters 1–2 notes

binary threshold:  I′(x,y) = 255 if I(x,y) ≥ T, else 0
auto-pick T:  θ = (μL + μH) / 2   // iterated until it stops moving

Thresholding turns a greyscale image into a binary one. The only decision is where to cut — and a well-separated histogram (two humps) makes that decision obvious. The valley between the humps is the natural threshold.

How to read a histogram

Two clear humps

A dark object on a light background (try Cells). The two peaks are object vs background; the valley between them is the ideal threshold. Easy, robust segmentation.

All bunched up

A low-contrast image piles every pixel into a narrow band — hard to threshold, washed-out to the eye. The fix is contrast stretching: spread that band back across 0–255.

Flat / spread out

A gradient touches every level roughly equally → a flat histogram. No natural valley, so a single global threshold cuts arbitrarily. This is when you reach for region methods instead.

🧠 Hook — "the dip is the threshold"

For a two-hump (bimodal) histogram, put T in the valley between the peaks. The iterative method θ=(μL+μH)/2 is just a way of sliding toward that valley automatically: split, take each side's mean, put T halfway, repeat.

Exam framing

What gets asked

"Sketch the histogram of this image and choose a threshold" or "describe an automatic threshold-selection method." Answer: a histogram is a count of pixels per intensity; for a bimodal histogram pick T in the valley; an automatic method iterates T = (μL + μH)/2 until T stops changing.

DON'T CONFUSE

The histogram throws away where pixels are — two very different images can share a histogram. It only describes the distribution of intensities, which is exactly why it's perfect for choosing a global threshold but useless for locating which pixel is which.
