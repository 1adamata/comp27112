Motivation

Why do we need transformations?

To build virtual worlds or overlay effects on real footage, we need to move objects around: make them bigger or smaller, spin them, flip them, slant them, and shift their position. In this chapter our test subject is the course's cow.

Every transformation below is just a rule for turning one vector into another. The magic of this chapter is that almost all of them can be written as a matrix multiplication — and matrices compose, so a whole chain of moves collapses into one matrix that a GPU can apply millions of times a second.

01 — Foundations

Linear transformations

A transformation T maps vectors to vectors: T : ℝⁿ → ℝᵐ. It is linear when it respects vector addition and scalar multiplication.

The three properties

1. Additivity: T(u + v) = T(u) + T(v)

2. Homogeneity: T(c·u) = c·T(u)

3. Origin fixed: T(0⃗) = 0⃗

Geometrically: grid lines stay straight, evenly spaced, and parallel, and the origin never moves. Keep that picture — it tells you instantly whether something is linear. (Spoiler: translation breaks rule 3.)

One matrix is enough

A matrix–vector product computes a new vector from the old one.

The key insight: just watch the basis vectors

Any vector is a combination of the basis vectors, v = Σ cᵢ b̂ᵢ. Because T is linear, T(v) = Σ cᵢ T(b̂ᵢ). So to know what a transform does to every vector, you only need to know what it does to the basis vectors — and those transformed basis vectors are exactly the columns of the matrix.

Mental modelA linear transformation doesn't just move one arrow — it morphs the entire grid. Sit with that idea; it's the heart of everything that follows.
02 — Scale

Scaling

Stretch or shrink the cow. Multiplying every coordinate by a scalar s works, but we prefer a matrix so it fits the same machinery as everything else.

The scale matrix is diagonal: values sit only on the main diagonal, zeros elsewhere.

S = [ sx 0 ; 0 sy ] → v′ = [ sx·vx ; sy·vy ]

For uniform scale, sx = sy. For non-uniform scale, set them independently — that single change is all you need, no extra machinery. Try it in the lab below.

03 — Rotation

Rotation

Spin a vector by an angle θ. Its length R never changes — only its direction does. We derive the new components from the action on the basis vectors.

Collecting the components into a matrix gives the rotation matrix. The minus sign decides the spin direction.

Watch a vector rotate

R(θ) =
cos θ−sin θ sin θcos θ
cos θ = 0.82   sin θ = 0.57

The columns of R are the rotated basis vectors î and ĵ — exactly the "watch the basis" idea in action.

Rotation matrices are orthogonal

Why it mattersThe two columns (sin/cos) are perpendicular unit vectors, so the matrix is orthogonal: its transpose equals its inverse. To undo a rotation you just transpose it — no expensive inverse needed. Rotating one way then the other (the transpose) returns you to the start: RᵀR = I.
04 — Reflection

Reflection

Mirror the cow across an axis. Reflections are also orthogonal — but they are not rotation matrices, so keep the distinction clear.

reflect across y: [ −1 0 ; 0 1 ] · reflect across x: [ 1 0 ; 0 −1 ] · across origin: [ −1 0 ; 0 −1 ]

Flip a sign on the diagonal and the corresponding axis is mirrored. Toggle the reflect chips in the lab to see the cow-shape flip.

05 — Shear / Skew

Shear

Slant the cow sideways. What is a shear, really? It's three transforms in disguise: rotate, scale non-uniformly, then rotate back.

This is the first hint of how powerful matrices are: a single "new" operation is built by composing ones we already have. Which raises a critical question — does the order of those matrices matter?

06 — Composition

Order matters

Combining transforms is "just" multiplying their matrices — but matrix multiplication is not commutative. Rotate-then-translate is a completely different result from translate-then-rotate.

Rotate & translate — in both orders

The same cow-shape, the same 40° rotation, and the same rightward translation — applied in two orders.

Rotate → Translate Translate → Rotate

Rotate → Translate: the shape spins in place, then slides right — it lands upright-rotated beside the origin.

Try toggling. Translate → Rotate swings the whole arm around the origin, throwing the shape onto a wide arc.

Read transforms right-to-left

The transform closest to the vector is applied first. So if we want to rotate (R1), then translate (T1), then scale (S1), then translate again (T2), the composite is read right→left:

Common bugGetting matrix order wrong is one of the most common errors in graphics engineering. Our shear from earlier had to flip its order too — R₂·S·R₁, not R₁·S·R₂ — or the cow shears the wrong way.
07 — Translation

Translation breaks the rules

Translation just adds an offset: v′ = v + d. Useful — e.g. to rotate the cow about a point P, move P to the origin, rotate, move back. But there's a problem.

Translation moves the origin, so it violates rule 3 (T(0⃗)=0⃗). It can't be written as a 2×2 matrix multiply, and it won't compose with rotation/scale the normal way. Stacking separate "multiply then add" steps a million times a second is exactly what we want to avoid.

Affine = linear + translation

A linear transformation plus a translation is an affine transformation (linear transforms are a special case where b = 0). Affine transforms keep grid lines straight, evenly spaced and parallel — but they're allowed to move the origin.

Exam noteThe course's MCQ answer for "an affine transform" is "will maintain the area of objects" (the other options are clearly wrong). Be precise in long answers though: affine maps preserve parallelism and ratios, and scale area by |det A| — area is only exactly preserved when |det A| = 1.
08 — The fix

Homogeneous coordinates

The trick that makes translation a matrix — discovered by August Ferdinand Möbius (yes, the strip). Add one extra coordinate and everything unifies.

Append a 1 to the vector and use a 3×3 matrix. The extra row lets the translation sit in the last column and still be a pure multiplication:

Position vs direction vectors

If the bottom entry is 1 you have a position; make it 0 and you have a pure direction — translation then has no effect on it, which is exactly what you want for things like surface normals.

The payoffNow every affine transform — scale, rotate, reflect, shear, translate — is a single 3×3 matrix. A whole chain composes into one composite matrix, and your GPU's tensor cores will love it.
09 — Into 3D

3D transformations

Good news: 3D is barely different. Add a z dimension and use 4×4 matrices. Scale and translate work the same way, just with an extra axis.

Rotation is the one that's different

In 2D we rotate about a point. In 3D we rotate about an axis, so there's a separate matrix for the x, y and z axes. Rotating about z looks identical to the 2D case — the z coordinates simply sit still.

Order, againDoing one axis at a time gives jagged motion; the smooth result comes from a composite matrix of all the rotations at once. And just like 2D, order still matters.
