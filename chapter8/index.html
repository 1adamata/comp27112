01 / FILE FORMATS

What lives inside an image file?

An image file is far more than a grid of pixels. The mix of things a format can hold is exactly why so many formats exist — each was designed around a different priority.

  • Image data (pixels) — greyscale or colour samples
  • Colour palette — a small lookup table of colours used
  • Different-size versions — e.g. icons at several resolutions
  • Transparency information — an alpha channel
  • Camera data — model, focal length, shutter speed, ISO…
  • Location — GPS latitude & longitude
Key ideaFormats differ by which of these they support, how they encode pixels, and whether they compress — lossy or lossless.

01 / IMAGE DATA

Bit depth: how much per pixel?

So far we've used an unsigned byte (8-bit, 0–255) for greyscale and 24-bit colour (three 8-bit channels) for RGB. But a pixel doesn't always need a whole byte — and sometimes a byte isn't enough.

1-bit — the Portable Bit Map (.pbm)

After thresholding, an image has only two values: black and white. Storing that as a full byte per pixel is wasteful — each pixel is really just on / off, one bit. The PBM format packs eight pixels into a single byte, giving an instant 8:1 saving.

Interactive · pack 8 pixels into 1 byte

Click pixels to toggle black (1) / white (0). Watch the byte form.

byte = 000000002  =  0x00  =  010

One byte → eight pixels. A 440×400 image = 176,000 pixels → just 22.1 kB as PBM (vs 176 kB as one byte per pixel).

02 / EXPOSURE — AN ASIDE

Controlling the light

How can one camera shoot a blazing sunny street and a dim room? It must vary how much light reaches the sensor. Two controls do this — and each has a side-effect on the picture.

Aperture — the iris

Interactive · open and close the aperture

f /f/8

Smaller f-number = larger hole = more light. So f/1.8 is wide open; f/32 is a pinhole. It's a ratio (focal length ÷ aperture diameter), which is why the numbers feel "backwards".

opening: 50% · light: medium

Compare it with the pupil of your eye, which widens in the dark and shrinks in bright light.

Real iris from an SLR lens, stepping from tiny to wide open.

Side-effect: depth of field

Aperture doesn't only change brightness — it changes how much of the scene is in focus. A small aperture (large f) keeps everything sharp; a large aperture (small f) blurs the background — a shallow depth of field.

Interactive · slide between f/32 and f/5.6

f/32 — deepf/5.6 — shallow

Same flower. At f/32 the lane behind stays sharp; at f/5.6 it melts away. This blur is optical — not an electronic phone-camera trick.

Shutter — the time control

The shutter sets how long light is collected. A long exposure lets in more light but lets moving things smear; a fast shutter freezes motion but lets in less light.

Interactive · slow shutter ↔ fast shutter

1/10 s1/640 s

Slow (1/10 s): the car smears across the frame. Fast (1/640 s): action frozen. (A slow shutter also blurs from your own hand-shake — hence tripods.)

The trade-offAperture & shutter are two ways to hit the same total light. On a windy day you need a fast shutter to freeze a flower — so you must open the aperture wider to compensate, which shrinks the depth of field. Every choice ripples. On the mode dial: A/Av = Aperture priority, S/Tv = Shutter (Time) priority.
SLR path. Light → mirror → pentaprism → viewfinder. Press the button: the mirror flips up (most of the "click") so light hits the sensor.

Exposing for the scene

The meter averages the scene (or a chosen point) and picks settings. But a high-contrast scene can't all be exposed correctly at once.

02 / IMAGE DATA — HDR

High Dynamic Range

HDR is about capturing a larger range of brightness than a normal 8-bit image can hold — so both the dark gravestone and the bright sky survive in one picture.

The crude version

Take two shots — one exposed for each subject — and paste the good parts together. The lecture's "botched" example below stitches the well-exposed gravestone into the well-exposed church shot.

Bracketing

Some cameras fire several frames at once, each at a different exposure (bracketing), then merge them. Better: capture everything in one image with a wider-range sensor.

  • Modern CMOS sensors reach higher dynamic range
  • RAW formats are often 12-bit/channel → 4096 levels (vs 256). RAW formats are proprietary per manufacturer.
  • Up-to-16-bit/channel formats: TIFFPNGJPEG2000
  • 24-bit = 8 bits/ch; 48-bit "deep colour" = 16 bits/ch. The HDMI standard allows 16 bits per channel.

01 / COLOUR PALETTE

Doing more with fewer bits

On older formats, embedded systems with little memory, or low-colour displays, we don't want to waste 24 bits per pixel. Two classic tricks: an indexed palette, and bit-packed colour.

GIF — an indexed palette

GIF stores only 8 bits per pixel, even for colour. Each pixel value is an index into a 256-entry palette of full 24-bit colours, saved inside the file. The palette is chosen per-image from its most common colours — so it differs for every picture.

Interactive · palette lookup

8-bit packed colour (3-3-2)

No palette needed — just split the 8 bits: 3 for red, 3 for green, 2 for blue. Popular on embedded LCD displays.

01 / MORE THAN PIXELS

Sizes, transparency & metadata

Multiple sizes in one file (.ico)

A Windows .ico file holds the same icon drawn at several resolutions, so the OS picks the right one (title-bar vs desktop). Why redraw rather than shrink? Shrinking by dropping rows can erase thin lines and ruin a small icon — so each size is hand-tuned.

Transparency — the alpha channel

An alpha channel says how "see-through" each pixel is. A JPEG (no alpha) lays an opaque white box over the scene; a PNG with alpha lets the background show through the deleted areas. And alpha isn't all-or-nothing — it's a smooth 0–100%.

Camera data & location (EXIF / GPS)

EXIF. Aperture, exposure, focal length, ISO, model, date — stored alongside the pixels. (Focal length is given both as the real lens value and as a 35 mm-film equivalent so it's comparable between cameras with different sensor sizes.)
GPS. Latitude & longitude let you map exactly where a photo was taken. A compass / IMU direction would make it even more useful (e.g. for drone imagery).

Run-Length Encoding

The simplest compression: replace runs of identical values with (count, value) pairs. It's lossless — the exact image can be rebuilt.

03 / COMPRESSION — JPEG

How JPEG throws data away

JPEG is lossy: the exact image can't be recovered. It works on 8×8 blocks, converts each to frequencies, then discards the high-frequency detail your eye barely notices.

Step through the pipeline

The DCT (Discrete Cosine Transform)

Each 8×8 block of pixels becomes an 8×8 grid of frequency coefficients. Low frequencies cluster at the top-left, highs at the bottom-right. Most of the energy ends up top-left.

F(u,v) = ¼·C(u)C(v)·ΣΣ f(x,y)·cos[(2x+1)uπ/16]·cos[(2y+1)vπ/16]
C(0)=1/√2, otherwise C=1  ·  (easy way: OpenCV cv::dct())

Quantise → zig-zag → RLE

Divide the DCT by a quantisation table (big numbers at the high frequencies). Many coefficients round to 0. Read the block in a zig-zag so all those zeros bunch together, then RLE-compress the run of zeros.

RLE: (1,79) (1,0) (1,-2) (3,-1) (2,0) (2,-1) (54,0)
DecodeTo rebuild: multiply by the same quantisation table (de-normalise), then run the inverse DCT. The pixels come back very close to the originals — even though most coefficients were thrown away.

Quality vs file size

The "quality" setting just controls how aggressively the quantisation table crushes coefficients. Higher quality = less compression = bigger file.

Lossless

Exact reconstruction. RLE (used in PCX). No information lost.

Lossy

JPEG. Can't be reconstructed exactly, but stays very close while shrinking a lot. The DCT step is just one part of the full JPEG algorithm.

SUMMARY

So why so many formats?

Ownership

Proprietary (RAW, kept secret) vs open standards anyone can implement. GIF's patented compression pushed people to JPEG & open formats in the '90s.

Colour

Greyscale vs colour; full colour vs palettes; different bit depths (1, 8, 24, 48-bit).

Growth

New needs keep arriving — transparency, location, camera info — and future formats will store things we don't even need yet.

In one lineFile formats trade off compatibility, size, colour fidelity, features and openness — no single format wins on all axes, so we keep them all around.
